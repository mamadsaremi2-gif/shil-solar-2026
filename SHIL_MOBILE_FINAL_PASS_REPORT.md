# SHIL Mobile Final Pass Report

## هدف این پکیج
این پکیج برای اصلاح نهایی ظاهر موبایل SHIL پس از تست Vercel و اسکرین‌شات‌های واقعی ساخته شد. تمرکز این مرحله روی حذف رفتارهای دسکتاپی، یکدست‌سازی ظاهری و تثبیت قانون Header / Rail / Content / Footer است.

## فایل مرجع اصلی
از این مرحله به بعد مرجع اصلی ظاهر موبایل همچنان این فایل است:

```txt
src/appearance/styles/shil-mobile-design-system.css
```

همچنین برای سازگاری با کامپوننت Frame، همان قوانین نهایی در این فایل نیز append شده‌اند:

```txt
src/components/ShilFrame.css
```

## تغییرات ساختاری

### 1. داشبورد
- Header از داشبورد حذف شد.
- Footer از داشبورد حذف شد.
- Glass یا پنل شیشه‌ای پشت آیکون‌ها حذف شد.
- آیکون‌ها مستقیم روی بک‌گراند قرار گرفتند.
- Grid داشبورد حالت 3 ستونه موبایلی دارد.
- سایه فقط روی خود تصویر آیکون اعمال شده، نه روی باکس پشت آن.

### 2. Shell داشبورد
فایل زیر اصلاح شد:

```txt
src/components/ShilPageShell.jsx
```

اکنون اگر مسیر `/dashboard` باشد، کامپوننت‌های Header و Footer اصلاً render نمی‌شوند؛ یعنی فقط با CSS مخفی نشده‌اند، بلکه از DOM داشبورد هم حذف شده‌اند.

### 3. Mobile First واقعی
- عرض اصلی اپ روی `100vw` تثبیت شد.
- محدودیت‌های دسکتاپی و max-widthهای مزاحم override شدند.
- overflow افقی در لایه‌های اصلی بسته شد.
- فضای خالی اطراف اپ در موبایل هدف‌گذاری و حذف شد.

### 4. Header صفحات مهندسی
- ارتفاع Header کوچک‌تر شد.
- Header کاملاً مینیمال شد.
- حالت شیشه‌ای شفاف با blur سبک اعمال شد.
- دکمه‌های Header کوچک‌تر و capsule شدند.
- عنوان Header به یک capsule مرکزی کوتاه و خوانا تبدیل شد.

### 5. Rail / مسیر مراحل
- Rail کوچک‌تر شد.
- Rail دقیقاً به زیر Header چسبیده است.
- فاصله بین Header و Rail حذف شد.
- Rail نیز glass و شفاف است.
- آیکون‌های داخل Rail دیگر پشت‌زمینه شیشه‌ای ندارند.
- آیکون‌های Rail کوچک‌تر و فشرده‌تر شدند.
- مرحله فعال با `is-active-step` و `aria-current="page"` مشخص می‌شود.

### 6. Active Step
فایل زیر اصلاح شد:

```txt
src/components/IosIconGrid.jsx
```

برای آیتم فعال، `aria-current="page"` اضافه شد تا CSS بتواند مرحله جاری را دقیق‌تر تشخیص دهد.

### 7. Scroll Content
- محتوای صفحات از زیر Header و Rail شروع می‌شود.
- هنگام اسکرول عمودی، دیتا زیر Rail می‌رود و با آن تداخل نمی‌سازد.
- Padding پایین صفحه بر اساس ارتفاع Footer تنظیم شد.
- دکمه‌های تأیید و submit دیگر زیر Footer نمی‌مانند.
- آخرین بخش صفحه چند میلی‌متر بالاتر از بالاترین نقطه Footer قابل مشاهده می‌ماند.

### 8. فاصله از دیواره موبایل
- فاصله کناری صفحات به `7px` موبایلی تنظیم شد.
- کارت‌ها، پنل‌ها، بلوک‌ها و فیلدها اجازه خروج از عرض صفحه ندارند.
- هیچ کارت یا ورودی نباید به دیواره بچسبد یا از دیواره بیرون بزند.

### 9. یکدست‌سازی رنگ فیلدها و بلوک‌ها
- رنگ کارت‌ها و فیلدها در صفحات مهندسی یکپارچه شد.
- رنگ‌بندی پراکنده سبز، بنفش، آبی و فیروزه‌ای کنترل شد.
- Field Theme واحد ایجاد شد:
  - کارت‌ها: روشن، مهندسی، نیمه‌شفاف
  - فیلدها: سفید تمیز با border فیروزه‌ای کنترل‌شده
  - متن‌ها: تیره و خوانا

### 10. بک‌گراندها
- بک‌گراند داشبورد از صفحات مهندسی جدا شد.
- صفحات مهندسی دیگر نباید تصویر یا بک‌گراند داشبورد را بخوانند.
- بک‌گراند مهندسی یکپارچه برای صفحات غیر داشبورد تعریف شد.

## وضعیت CSSهای قدیمی
CSSهای قدیمی و Hotfixها از پروژه حذف فیزیکی نشده‌اند تا ریسک شکستن صفحات پایین بماند، اما ورودی اصلی اپ فقط Design System مرکزی است. قوانین نهایی در انتهای فایل مرکزی قرار گرفته‌اند تا Legacy CSSها را override کنند.
