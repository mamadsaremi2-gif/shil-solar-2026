# SHIL Managed Release V15.1

## هدف این نسخه
این نسخه برای مدیریت صحیح پروژه و پایدارسازی اجرای اولیه آماده شده است، بدون حذف یا تخریب هسته اصلی اپ، موتور محاسبات، داده ها، مسیرهای طراحی، UI اصلی، دارایی ها و زیرساخت های آینده.

## تغییرات انجام شده

### 1. بازیابی فایل ورودی اصلی اپ
فایل `index.html` که برای اجرای Vite ضروری است به ریشه پروژه اضافه شد. نبود این فایل باعث می شد Build با خطای `Could not resolve entry module index.html` متوقف شود.

### 2. ایمن سازی Bootstrap اپ
فایل `src/main.jsx` بازنویسی شد تا ابتدا رابط اصلی و صفحه ورود Render شود و سرویس های جانبی بعد از آن به صورت ایمن اجرا شوند.

سرویس های زیر از مسیر اجرای اجباری خارج شدند و در Bootstrap اختیاری قرار گرفتند:
- شبیه ساز Realtime
- Web Vitals و Preload Asset
- Global Error Handlers
- Sentry و Telemetry سازمانی

این کار مانع از آن می شود که خطای یک سرویس جانبی باعث بالا نیامدن Login یا کل اپ شود.

### 3. اضافه شدن Error Boundary سطح ریشه
اپ داخل `ProductionErrorBoundary` قرار گرفت تا خطاهای Runtime کنترل شوند و صفحه سفید کامل ایجاد نشود.

### 4. اضافه شدن Bootstrap اختیاری
فایل جدید `src/app/bootstrapOptionalServices.js` اضافه شد. این فایل سرویس های غیرحیاتی را با `dynamic import` اجرا می کند و در صورت خطا، اپ اصلی را متوقف نمی کند.

### 5. اضافه شدن ابزار اعتبارسنجی پروژه
فایل `tools/validate-project.mjs` اضافه شد و اسکریپت زیر در `package.json` ثبت شد:

```bash
npm run validate
```

این ابزار موارد زیر را بررسی می کند:
- وجود فایل های اصلی پروژه
- صحت Importهای نسبی داخل `src`
- جلوگیری از شکست های ساده قبل از Build

### 6. اصلاح اسکریپت های NPM
اسکریپت های `dev`، `build` و `preview` به شکل استاندارد Vite اصلاح شدند تا وابسته به مسیر داخلی `node_modules/vite/bin/vite.js` نباشند.
