# SHIL V15 UI Standardization Pack

## هدف
قفل کردن استانداردهای ظاهری و UX تمام مسیرهای اصلی و فرعی اپ SHIL از Dashboard تا خروجی نهایی.

## هویت طراحی
SHIL یک پلتفرم مهندسی است؛ نه فروشگاهی، نه مالی و نه کاتالوگی.

اصول:
- Industrial
- Engineering-focused
- Mobile-first
- Compact
- Runtime-aware
- Production-ready

## صفحات استانداردشده
- Dashboard
- ارتباط با ما
- سناریوهای آماده
- زیرشاخه‌های سناریوهای آماده
- پروژه‌ها
- پروژه جدید / صفحه مادر 3×3
- اطلاعات پروژه
- شرایط محیطی
- انتخاب مسیر پروژه
- روش محاسبات
- ورودی محاسبات
- تنظیمات سیستم
- چکیده اطلاعات
- اجرای محاسبات
- خروجی نهایی

## قوانین Background
- بکگراند اصلی SHIL از زیرساخت مدیریت می‌شود.
- کاربر بکگراند را تغییر نمی‌دهد.
- آفلاین: آخرین بکگراند کش‌شده نمایش داده می‌شود.
- آنلاین: Silent Update انجام می‌شود.
- Overlay تیره ثابت برای خوانایی الزامی است.

## قوانین Header
### صفحات عمومی
Back/Dashboard + Centered SHIL Brand + Capsule Title

### زیرشاخه‌های پروژه جدید
Dashboard Button + Centered SHIL Brand + Dynamic Stage Title

اگر کاربر از دکمه Dashboard خارج شود:
Auto Save انجام شده و پروژه در پروژه‌های در حال اجرا ذخیره می‌شود.

## قوانین Footer
### ارتباط با ما
Single Dashboard Button

### صفحه مادر پروژه جدید
Single Dashboard Button

### تمام مراحل پروژه جدید
Right: مرحله قبل  
Center: ذخیره پیش‌نویس  
Left: تأیید مرحله / ادامه

این Footer تا انتهای پروژه جدید ثابت است.

## قوانین Scroll
- اسکرول عمودی صفحه فعال است.
- اسکرول افقی سراسری ممنوع است.
- اسکرول افقی داخلی فقط برای بلوک‌های دیتاسنگین مجاز است.
- هر بلوک افقی باید نوار اسکرول دائمی زیر خودش داشته باشد.

## قوانین بلوک‌ها
بلوک‌ها باید Elevated Industrial Panels باشند:
- برجسته و بیرون‌زده از بکگراند
- تفاوت رنگ ملایم با بکگراند
- Neon Border نرم
- Shadow کنترل‌شده
- Matte Dark Overlay
- Padding فشرده
- Scroll Indicator برای بلوک‌های افقی

## قوانین Input
- تمام فیلدها مقدار پیش‌فرض منطقی دارند.
- تمام فیلدها قابل Override دستی هستند.
- فارسی و انگلیسی در متن پشتیبانی می‌شود.
- اعداد فارسی و انگلیسی پشتیبانی می‌شود.
- قبل از محاسبات، ورودی‌ها Normalize می‌شوند.
- اولویت داده‌ها:
  1. User Override
  2. Online Data
  3. GPS / City Data
  4. Default Standard Values

## موتورهای محاسبات
دو موتور مستقل داریم:
- Solar Unified Calculation Engine
- Emergency Power Unified Calculation Engine

Calculation Dispatcher بر اساس مسیر پروژه موتور مناسب را انتخاب می‌کند.

## سناریوهای آماده
سناریوهای آماده
├── سناریوهای آماده انرژی خورشیدی
│   ├── سبک
│   ├── متوسط
│   └── سنگین
└── سناریوهای آماده برق اضطراری
    ├── سبک
    ├── متوسط
    └── سنگین

### مسیر خورشیدی آماده
انتخاب سناریو → شرایط محیطی → لیست تجهیزات → Solar Engine → خروجی نهایی

### مسیر برق اضطراری آماده
انتخاب سناریو → لیست تجهیزات → Emergency Engine → خروجی نهایی

## پروژه جدید
Dashboard → پروژه جدید → صفحه مادر 3×3 → اطلاعات پروژه → شرایط محیطی → انتخاب مسیر پروژه → روش محاسبات → ورودی محاسبات → تنظیمات سیستم → چکیده اطلاعات → اجرای محاسبات → خروجی نهایی

## وضعیت پروژه‌ها
اگر پروژه در هر مرحله رها شود:
Projects > پروژه‌های در حال اجرا

اگر پروژه تا خروجی نهایی کامل شود:
Projects > پروژه‌های نهایی

## شرایط محیطی
نقشه ایران، نقشه آنلاین، عکس محل نصب و قطب‌نما باید در Smart Visual Engineering Blocks قرار بگیرند، نه تصویر خام.

## خروجی نهایی
- عکس تجهیزات حذف می‌شود.
- هزینه حذف می‌شود.
- کارت شارژ کنترلر مستقل حذف می‌شود.
- MPPT داخل کارت اینورتر نمایش داده می‌شود.
