# SHIL Clean Architecture Reset

## کار انجام‌شده
- موتور محاسبات و Rule Engine از همه Gatewayهای اصلی غیرفعال شد.
- فایل‌های بکاپ/تکراری که در مسیر src بودند حذف شدند.
- رجیستری مرکزی قوانین در `src/rules` ساخته شد.
- نقشه پارتیشن‌ها در `src/partitions` اضافه شد.
- صفحه تنظیمات سیستم بدون وابستگی به موتور محاسبات باقی مانده تا کرش تکرار نشود.

## مسیر افزودن قوانین جدید
1. قانون جدید را فقط در `src/rules` تعریف کن.
2. قانون را به پارتیشن مرتبط در `SHIL_RULE_PARTITIONS` وصل کن.
3. اول با `enabled: false` تست UI بگیر.
4. سپس قانون را در یک Gateway کوچک و مستقل فعال کن.

## ممنوعیت‌ها
- قانون محاسباتی داخل صفحه React ننویسید.
- خروجی موتور را مستقیم در JSX مصرف نکنید؛ اول normalize شود.
- فایل backup داخل src نگه ندارید.
