# SHIL Partition Map

این نسخه برای نگهداری آسان پارتیشن‌بندی شده است. قوانین محاسباتی از UI جدا و فعلاً غیرفعال شده‌اند.

## پارتیشن‌ها
- `login`: صفر تا صد ورود، احراز هویت، نقش‌ها و Guardها
- `admin`: پنل ادمین و مدیریت تنظیمات مدیریتی
- `dashboard`: داشبورد، ویجت‌ها و وضعیت کلی پروژه‌ها
- `new-project`: مسیر پروژه جدید و مراحل ورود اطلاعات
- `project-settings`: صفحه پیکربندی/تنظیمات سیستم و بانک تجهیزات
- `calculation`: محل آینده موتور محاسبات و Rule Engine
- `summary`: چکیده، خروجی PDF/تصویر و اشتراک‌گذاری
- `shared`: کامپوننت‌ها، استایل‌ها، Assetها و ابزارهای مشترک

## قانون نگهداری
از این به بعد هیچ قانون محاسباتی نباید داخل صفحات React نوشته شود. همه قوانین باید از `src/rules` شروع شوند و سپس به پارتیشن مربوط وصل شوند.
