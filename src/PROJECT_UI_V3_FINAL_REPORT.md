# گزارش نهایی یکپارچه‌سازی صفحات پروژه — UI V3

## هدف
جایگزینی ظاهرهای قدیمی و متفاوت صفحات زیرمجموعه پروژه با یک زبان طراحی واحد، بدون تغییر در منطق محاسبات، مسیرها، داده‌ها و متن فارسی.

## تغییرات اجرایی
- ایجاد stylesheet واحد `appearance/styles/shil-project-unified-v3.css` و بارگذاری آن به‌عنوان آخرین لایه استایل.
- خارج کردن اسکریپت‌های MutationObserver قدیمی از مسیر اجرا؛ این اسکریپت‌ها به‌صورت runtime استایل inline و ناسازگار تزریق می‌کردند.
- خارج کردن CSSهای صفحه‌ای قدیمی مربوط به ProjectPath، ProjectInfo و Environment از `main.jsx`.
- خارج کردن cleanup و design-system قبلی از مسیر نهایی صفحات پروژه تا دو نسخه ظاهری هم‌زمان اجرا نشوند.
- حذف فایل‌های backup و نسخه‌های موقت صفحات از بسته نهایی.

## استانداردهای واحد اعمال‌شده
- فونت فارسی: Vazirmatn/Vazir با fallback امن Tahoma.
- جهت صفحه: RTL در کل پوسته مهندسی؛ رشته‌های فنی LTR به‌صورت isolate.
- عرض محتوای یکسان: حداکثر 920px.
- Header، Footer و Step Rail با اندازه و فاصله ثابت.
- عنوان بخش‌ها: ارتفاع، رنگ، فونت، border و radius یکسان.
- کارت‌ها و بلوک‌های اطلاعاتی: سطح سفید، border خنثی، shadow سبک و radius یکسان.
- فرم‌ها، input، select و textarea با ارتفاع و تایپوگرافی یکسان.
- جدول‌ها با line، header، padding، اندازه فونت و تراز یکسان.
- دکمه‌های اصلی و ثانویه با ارتفاع و radius یکسان و حذف transform/position قدیمی.
- هشدارها با یک الگوی رنگ و فاصله ثابت.
- Responsive واحد برای موبایل و تبلت.

## ایمنی و سازگاری
- JSX و منطق محاسبات تغییر نکرده است.
- متن‌های فارسی بازنویسی یا تبدیل encoding نشده‌اند.
- فایل‌های ویرایش‌شده با UTF-8 ذخیره شده‌اند.
- scope اصلی CSS فقط `.shil-engineering-shell` است؛ صفحات Login و Dashboard تحت تأثیر قرار نمی‌گیرند.
- استایل‌های پویا و داده‌محور حذف نشده‌اند.
