# گزارش اصلاح ایمن جریان پروژه جدید — V5.22

## هدف
حذف وابستگی باقی‌مانده به صفحه آیکون‌دار قدیمی، بدون تغییر ساختار صفحات ریل‌دار یا مسیرهای مهندسی موجود.

## اصلاحات انجام‌شده
- لینک آیکون «پروژه جدید» در داشبورد مستقیماً به `/new-project/path` متصل شد.
- لینک نوار پایین، فوتر و خلاصه پروژه به مسیر جدید هماهنگ شد.
- مسیر قدیمی `/new-project` حذف نشد؛ برای سازگاری با لینک‌ها و بوکمارک‌های قدیمی به‌صورت Redirect امن به `/new-project/path` باقی ماند.
- Import بلااستفاده صفحه قدیمی `NewProject` از Router حذف شد.
- ذخیره خودکار و Toast روی صفحه شروع `/new-project/path` متوقف شد تا پروژه خالی ساخته نشود.
- نسخه تکراری `src/src` حذف نشد تا ساختار بسته آسیب نبیند؛ همان اصلاحات در آن نیز Mirror شد تا دو نسخه ناسازگار باقی نماند.

## بخش‌هایی که عمداً تغییر نکردند
- مسیرهای صفحات ریل‌دار و ترتیب مراحل
- Routeهای execution و domain
- Guardهای مهندسی
- داده‌های پروژه و localStorage موجود
- ظاهر و CSS صفحات

## جریان نهایی
`Dashboard → /new-project/path → انتخاب مسیر → ادامه صفحات ریل‌دار`

مسیر قدیمی `/new-project` فقط نقش Redirect سازگار را دارد و دیگر صفحه مستقلی رندر نمی‌کند.
