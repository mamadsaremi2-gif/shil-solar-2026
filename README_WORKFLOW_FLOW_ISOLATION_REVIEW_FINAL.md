# SHIL Workflow Flow Isolation Review Final Update

این آپدیت بعد از بررسی مسیر «جداسازی کامل Flow پروژه جدید / سناریوی آماده / مسیر نیروگاهی» اعمال شد.

## اصلاحات تکمیلی

- ایزوله‌سازی رسمی سه حالت کاری:
  - پروژه جدید دستی
  - سناریوی آماده
  - مسیر مستقل نیروگاهی
- جلوگیری از ورود پروژه جدید معمولی به لیست تجهیزات سناریو بعد از شرایط محیطی.
- پاک‌سازی State سناریو هنگام شروع پروژه جدید یا انتخاب مسیر دستی.
- سناریوی آماده فقط در صورتی به لیست تجهیزات می‌رود که از صفحه سناریو آمده باشد و Scenario ID معتبر داشته باشد.
- مسیر برق اضطراری دستی دیگر مستقیم به Summary نمی‌رود و وارد Method/Equipment خودش می‌شود.
- مسیر نیروگاهی فقط از کارت مستقل نیروگاهی یا هشدار هوشمند توان بالای ۳۰kW فعال می‌شود.
- Route Guard اضافه شد تا Deep Link اشتباه به تجهیزات سناریو یا Utility Gateway کنترل شود.
- State Utility و Scenario از هم جدا شدند تا نشتی Flow بین مسیرها کمتر شود.

## رفتار نهایی مورد انتظار

```txt
پروژه جدید دستی
→ اطلاعات پروژه
→ شرایط محیطی
→ انتخاب مسیر پروژه
→ روش محاسبات/ورودی‌ها
→ تنظیمات سیستم
→ چکیده
→ اجرا
```

```txt
سناریوی آماده خورشیدی / برق اضطراری
→ انتخاب سناریو
→ شرایط محیطی
→ لیست تجهیزات همان سناریو
→ تنظیمات سیستم همان شاخه
→ چکیده
→ اجرا
```

```txt
نیروگاهی
→ کارت مستقل نیروگاهی
→ Utility Gateway
→ تنظیمات و محاسبات نیروگاهی
```

اگر در مسیر خورشیدی معمولی توان پنل‌ها از ۳۰kW بیشتر شود، اپ ادامه مسیر را متوقف و کاربر را به درگاه مستقل نیروگاهی هدایت می‌کند.
