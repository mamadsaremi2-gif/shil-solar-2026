# گزارش نسخه V16 UI Rebuild

این نسخه با هدف بازسازی لایه ظاهری SHIL ساخته شد، بدون حذف منطق پروژه، موتور محاسبات، Routeها، State و سرویس‌ها.

## تغییرات اصلی

- اضافه شدن فایل مرکزی UI جدید:
  - `src/styles/v16-ui-rebuild.css`
- اتصال این فایل به ورودی مرکزی ظاهر:
  - `src/styles/modular-appearance.css`
- تعریف Design Tokens جدید:
  - رنگ‌ها
  - Radius
  - Shadow سبک
  - Typography
  - Card system
  - Button system
  - Input system
  - Header / Footer behavior
- یکپارچه‌سازی هدرهای داخلی بدون حذف ساختار JSX موجود:
  - `v15-flow-header`
  - `flow-top-card`
  - `compact-page-header`
  - `scenario-flow-header`
  - `mobile-fixed-header`
- حفظ نقش خاص داشبورد، اما هماهنگ‌سازی با زبان بصری جدید.
- فعال‌سازی اسکرول عمودی در صفحات سنگین.
- فعال‌سازی اسکرول افقی برای مسیر مراحل و Stepperها.
- تک‌ستونه کردن بخش‌های دیتاسنگین در موبایل.
- اصلاح فرم‌ها، کارت‌ها، دکمه‌ها و گزارش‌ها به سبک سفید، صنعتی و مینیمال.

## مسیر لوگوی هدر

لوگوی هدر داخلی در مسیر زیر قرار دارد و کاربر می‌تواند آن را جایگزین کند:

```text
public/assets/brand/shil-logo-header.webp
```

## نکته مهم

این نسخه یک UI Rebuild کنترل‌شده است؛ یعنی CSSهای قدیمی حذف فیزیکی نشده‌اند، اما لایه V16 در آخرین مرحله Load می‌شود و ظاهر نهایی را کنترل می‌کند. این کار ریسک خراب شدن منطق اپ را کاهش می‌دهد.
