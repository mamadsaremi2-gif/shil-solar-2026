# SHIL Mobile Data Table Final Update

## هدف آپدیت
این آپدیت قانون جدید نمایش داده‌ها را روی نسخه موبایل SHIL اعمال می‌کند: داده‌های کوتاه و عددی دیگر نباید در فیلدهای بزرگ و جداگانه نمایش داده شوند؛ بلکه باید در سطرهای فشرده Property / Value یا جدول‌های کوچک داخلی قرار بگیرند.

## تغییرات اصلی

### 1. قانون Property / Value
- داده‌های کوتاه مثل شهر، PSH، توان، جریان، مختصات و وضعیت‌ها به ردیف‌های فشرده تبدیل شدند.
- ارتفاع فیلدهای Legacy کاهش پیدا کرد.
- فاصله بین بلوک‌ها به چند میلی‌متر محدود شد.

### 2. GPS و داده‌های چندتایی
- کنترل‌های GPS و مختصات به Mini Table داخلی تبدیل شدند.
- دکمه‌های روش ثبت GPS فشرده شدند.
- Lat / Long کنار هم نمایش داده می‌شوند تا فضای عمودی کمتر مصرف شود.

### 3. کارت‌های انتخاب مسیر پروژه
- کارت‌های «پنل خورشیدی» و «برق اضطراری» از حالت عکس + متن + فضای سفید جدا خارج شدند.
- خود تصویر حالا کارت را تشکیل می‌دهد.
- عنوان، توضیح، نشانگر انتخاب و Overlay روی تصویر قرار گرفتند.
- فضای پرت کارت‌ها به شکل جدی کاهش پیدا کرد.

### 4. فاصله از دیواره موبایل
- Safe Edge واحد برای صفحات مهندسی اعمال شد.
- کارت‌ها و داده‌ها نه به دیواره می‌چسبند و نه از عرض موبایل بیرون می‌زنند.

### 5. یکدست‌سازی رنگ فیلدها
- رنگ‌های متنوع و پراکنده فیلدها با تم واحد روشن/مهندسی کنترل شدند.
- کارت‌ها، فیلدها، جدول‌ها و بلوک‌های داده از یک خانواده رنگی تبعیت می‌کنند.

### 6. کاهش اسکرول عمودی
- با تبدیل فیلدهای بزرگ به ردیف‌های فشرده، ارتفاع بسیاری از صفحات کاهش پیدا می‌کند.
- در صفحات اطلاعات پروژه، شرایط محیطی، GPS و نتایج محاسبات، داده‌های بیشتری در یک View دیده می‌شوند.

## قانون ادامه توسعه
از این نسخه به بعد، هر داده کوتاه یا عددی در SHIL باید در قالب زیر نمایش داده شود:

```txt
Property | Value
```

و فقط داده‌های قابل ویرایش یا متن‌های بلند مجاز هستند به صورت Input یا Textarea بزرگ نمایش داده شوند.
