# گزارش نسخه V18 — Enterprise Workspace UI

## هدف نسخه
V18 برای تکمیل مسیر V17 ساخته شد: حفظ Header/Footer/Shell پایدار، اما ارتقای تجربه کاربری به یک محیط موبایل مهندسی، سبک‌تر، قابل کنترل‌تر و نزدیک‌تر به زبان بصری iOS 26.

## تغییرات انجام‌شده

### 1. تثبیت Mobile Frame
- حداکثر عرض اپ برای موبایل هوشمند روی 430px کنترل شد.
- خروج محتوای صفحه از چپ/راست محدود شد.
- بدنه اصلی اپ برای موبایل واقعی، PWA و Safari بهینه‌تر شد.

### 2. Header و Footer مرکزی
- Header ثابت V17 حفظ و دقیق‌تر شد.
- Capsule عنوان صفحه جمع‌وجورتر شد.
- لوگوی مرکزی در جای ثابت خود باقی می‌ماند.
- Footer ثابت برای صفحات داخلی حفظ شد.
- محتوا هنگام اسکرول زیر Header/Footer مخفی می‌شود.

### 3. Scroll Architecture
- اسکرول عمودی برای همه صفحات داخلی فعال ماند.
- اسکرول افقی فقط برای بلوک‌های محلی مثل جدول، Stepper، تب‌ها، چیپ‌ها و اکشن‌های پروژه فعال شد.
- از اسکرول افقی سراسری جلوگیری شد.

### 4. Dashboard ساده‌تر و موبایلی‌تر
- داشبورد به‌عنوان صفحه مستقل حفظ شد.
- Hero داشبورد به نسبت موبایلی فول‌اسکرین‌تر نزدیک شد.
- کارت‌های داشبورد به Grid سه‌ستونه بانکی‌تر تبدیل شدند.
- Glow/Neon سنگین داشبورد کاهش پیدا کرد.
- کارت پروژه‌ها در داشبورد سبک‌تر و خواناتر شد.

### 5. New Project / 9 Icon Grid
- Grid 3×3 برای صفحه شروع پروژه و مسیرهای پروژه استانداردتر شد.
- کارت‌ها مربعی، فشرده، بانکی و مناسب فاصله بین Header/Footer شدند.

### 6. فرم‌ها و داده‌های سنگین
- کارت‌ها، فیلدها، دکمه‌ها و جداول به زبان ظاهری روشن و تمیزتر نزدیک شدند.
- جدول‌ها حداقل عرض داخلی و اسکرول افقی محلی گرفتند.
- Header جدول‌ها Sticky شدند.

### 7. Image Zones
- محل قرارگیری تصاویر مهم استاندارد شد:
  - Hero/Dashboard: نسبت 16:9 برای صفحات داخلی و 9:16 برای داشبورد موبایل
  - Contact: نسبت 4:5
  - Route Cards: نسبت 4:3
  - Iran Map: نسبت 16:10
