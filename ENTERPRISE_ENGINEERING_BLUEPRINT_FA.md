# SHIL SOLAR - نسخه مهندسی‌محور Enterprise Blueprint

## اصل معماری

این نسخه بر پایه «مهندسی‌محور بودن» توسعه داده شده است، نه فروش‌محور بودن. بانک تجهیزات SHIL در این نسخه نقش دیتابیس پارامترهای طراحی دارد و به عنوان فروشگاه یا سبد خرید استفاده نمی‌شود.

## تغییرات اصلی

### 1. Unified Engineering Engine
- مسیر ساده و حرفه‌ای از یک موتور مشترک استفاده می‌کنند.
- کاربر ساده فقط ورودی کمتر می‌دهد، اما خروجی همان خروجی مهندسی معتبر است.
- خروجی شامل `engineMeta` است تا مشخص شود محاسبات Offline-first و Cloud-enhanced هستند.

### 2. Engineering Equipment Bank
- بانک پنل، اینورتر، سانورتر و باتری SHIL اضافه شد.
- این بانک شامل پارامترهای مهندسی مثل توان، ولتاژ، MPPT، Surge، Parallel Capability، DoD و راندمان است.
- هدف بانک تجهیزات، انتخاب و اعتبارسنجی مهندسی است، نه فروش.

### 3. Parallel Inverter Logic
- موتور اکنون می‌تواند تعداد اینورتر لازم را بر اساس توان دائم و Surge محاسبه کند.
- در Hybrid و Offgrid فقط مدل‌های دارای قابلیت پارالل/سنکرون باید استفاده شوند.
- در Ongrid امکان چند اینورتر مستقل/موازی AC لحاظ شده است.
- در صورت کافی نبودن سقف پارالل، Validation خطای توقف می‌دهد.

### 4. PV Distribution Per Inverter / MPPT
- وقتی چند اینورتر لازم باشد، تعداد MPPT کل برابر تعداد MPPT هر واحد ضربدر تعداد اینورترها می‌شود.
- خروجی `pvDistributionByInverter` نشان می‌دهد پنل‌ها روی هر اینورتر و MPPT چگونه پخش شده‌اند.

### 5. Engineering Validation Layer
- خطاهای حیاتی مثل کمبود توان پارالل اینورتر، خطای ولتاژ، افت کابل، کمبود باتری و خطای MPPT در خروجی Validation ثبت می‌شود.
- برای هر خطا علت و پیشنهاد اصلاح ارائه می‌شود.

### 6. Management Audit
- خروجی `engineeringAudit` اضافه شد.
- هشدارها و خطاهای حیاتی آماده انتقال به کارتابل مدیریت هستند.
- این بخش می‌تواند در Supabase table مانند `engineering_audit_events` ذخیره شود.

### 7. Site Survey / Shadow Ready Input
- ساختار ورودی Site Survey شامل GPS، قطب‌نما، زاویه، عکس محل و اسکرین‌شات قطب‌نما اضافه شد.
- ساختار Shadow Objects برای موانع، ارتفاع، فاصله، جهت و ساعات بحرانی آماده شد.

### 8. Temperature Correction Readiness
- ضرایب دمایی Voc، Vmp، Isc و توان پنل از هم جدا شدند تا در فاز بعد محاسبات دقیق‌تر اعمال شوند.

### 9. Volume Management
- تصویرهای بزرگ اصلی فشرده شدند.
- نسخه تکراری نقشه محیطی حذف شد.
- فایل‌های runtime مهم حفظ شدند.

## نکات باقی‌مانده برای فاز بعد

1. UI کامل Site Survey و Shadow Analysis باید ساخته شود.
2. اتصال NASA/Solcast باید با کش آفلاین انجام شود.

4. جدول Supabase برای `engineering_audit_events` پیشنهاد می‌شود.
5. مهاجرت به TypeScript همچنان اولویت زیربنایی است.
