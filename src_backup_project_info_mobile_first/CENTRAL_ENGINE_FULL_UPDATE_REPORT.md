# SHIL Central Engineering Update Report

## هدف آپدیت
این نسخه برای حذف اصلاحات پراکنده صفحه‌ای و ایجاد یک جریان داده واحد بین ورودی محاسبات، تنظیمات سیستم، چکیده و خروجی نهایی آماده شده است.

## تغییرات اصلی اعمال‌شده

### 1. State مرکزی طراحی
فایل جدید اضافه شد:

```txt
src/engineering/core/projectDesignState.js
```

این فایل وضعیت نهایی طراحی را در کلید زیر ذخیره و بازیابی می‌کند:

```txt
shil:projectDesignState
```

صفحات بعدی به جای حدس‌زدن دوباره، از همین خروجی تاییدشده استفاده می‌کنند.

### 2. اصلاح انتقال خودکفایی
در `CalculationInputs.jsx` منطق خودکفایی اصلاح شد:

- `ساعت خودکفایی = 0` دیگر به `24` تبدیل نمی‌شود.
- `روز خودکفایی = 1` مستقل ذخیره می‌شود.
- مقدار کل ساعت فقط برای محاسبات داخلی نگهداری می‌شود، نه برای جایگزینی فیلد ساعت.

### 3. اصلاح موتور خورشیدی
در `src/engineering/solar/solarDesignEngine.js`:

- مقدار صفر برای ساعت خودکفایی معتبر شناخته می‌شود.
- سناریوی `ongrid` باتری را غیرفعال می‌کند.
- ضریب افزایشی/کاهشی از حالت دوگانه ناسازگار خارج شد.
- محاسبه باتری بر اساس روز/ساعت ثبت‌شده انجام می‌شود.
- توان‌های بالای 10kW دیگر سراغ Utility On-grid نمی‌روند و برای مسیر Off-grid/Hybrid ترکیب اینورترهای پارالل پیشنهاد می‌شود.

### 4. اصلاح انتخاب اینورتر
در `solarBankRules.js`:

- برای توان‌های بالاتر از 10kW، انتخاب به سمت اینورترهای قابل پارالل تا 8kW هدایت شد.
- نمونه 24kW حالا به جای Utility 30kW، به سمت `3 × 8kW` می‌رود.

### 5. اصلاح صفحه تنظیمات سیستم
در `SystemSettings.jsx`:

- فیلد اضافه و ناسازگار «مقدار ضریب» حذف شد.
- فقط «نوع ضریب طراحی» باقی ماند.
- ضریب اعمال‌شده به صورت read-only نمایش داده می‌شود.
- بانک‌ها خلوت شدند و فقط مشخصات خلاصه نشان می‌دهند.
- دیتاشیت‌ها همچنان داخل دکمه «نمایش جزییات دیتاشیت» قابل مشاهده‌اند.
- چکیده تنظیمات سیستم کامل‌تر شد و موارد زیر را نشان می‌دهد:
  - توان طراحی با ضریب
  - انرژی طراحی روزانه با ضریب
  - پنل معرفی‌شده
  - توان کل پنل‌ها
  - اینورتر معرفی‌شده
  - استرینگ پیشنهادی
  - تولید روزانه تخمینی
  - باتری معرفی‌شده

### 6. اصلاح چکیده و خروجی نهایی
`SummaryPage.jsx` و `RunCalculation.jsx` تا حد ممکن به `projectDesignState` متصل شدند تا خروجی نهایی از همان طراحی تاییدشده صفحه تنظیمات سیستم خوانده شود.

### 7. اعداد انگلیسی
در بخش‌های اصلی اصلاح‌شده، نمایش عددها به انگلیسی استاندارد شد و استفاده مستقیم از تبدیل فارسی در خروجی نهایی حذف شد.

## مواردی که هنوز نیاز به تست کاربر دارند

- صحت کامل مسیرهای غیرخورشیدی با دیتای اختصاصی خودشان.
- استایل نهایی کارت‌های بانک‌ها در اندازه واقعی موبایل.
- کنترل نهایی همه خروجی‌های PDF/Image برای نبود عدد فارسی.
- طراحی حفاظت و کابل هنوز در سطح پایه است و نیاز به فاز بعدی دارد.
