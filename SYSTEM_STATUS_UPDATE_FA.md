# گزارش تغییر وضعیت سامانه داشبورد

## هدف
بخش «وضعیت سامانه» در داشبورد قبلاً یک متن ثابت بود و همیشه مقدار «فعال» را نشان می‌داد. در این نسخه به یک وضعیت واقعی و پویا تبدیل شد.

## فایل‌های تغییر یافته

- `src/features/dashboard/DashboardPage.jsx`
- `src/styles/globals.css`

## رفتار جدید

داشبورد حالا موارد زیر را بررسی می‌کند:

1. آنلاین یا آفلاین بودن مرورگر کاربر از طریق `navigator.onLine`
2. فعال بودن حالت آفلاین داخلی اپ از طریق `AuthProvider`
3. قابل استفاده بودن `localStorage` برای ذخیره پروژه‌ها روی دستگاه
4. تنظیم بودن یا نبودن Supabase از طریق envهای پروژه
5. تلاش برای بررسی اتصال Supabase در صورت آنلاین بودن و تنظیم بودن cloud

## حالت‌های قابل نمایش

### آنلاین
وقتی اینترنت فعال است، Supabase تنظیم شده و ارتباط cloud تأیید می‌شود.

### آفلاین
وقتی مرورگر آفلاین است یا کاربر وارد حالت آفلاین اپ شده است.

### محلی
وقتی ذخیره محلی فعال است، ولی Supabase برای پروژه تنظیم نشده است.

### سرور نامطمئن
وقتی اینترنت فعال است، ولی اتصال cloud تأیید نمی‌شود. در این حالت ذخیره محلی همچنان فعال است.

### خطای ذخیره
وقتی مرورگر اجازه استفاده از `localStorage` را ندهد یا ذخیره محلی در دسترس نباشد.

## نکته مهم
در این تغییر، ذخیره‌سازی پروژه‌ها همچنان بدون وابستگی به cloud کار می‌کند. اگر cloud یا اینترنت مشکل داشته باشد، داشبورد این وضعیت را به کاربر اعلام می‌کند و اپ همچنان از ذخیره محلی استفاده می‌کند.

## تست

`npm run build` اجرا شد و build موفق بود.

هشدار باقی‌مانده مربوط به حجم chunkهای نهایی است و به این تغییر ارتباطی ندارد.
