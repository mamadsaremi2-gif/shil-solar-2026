# SHIL Solar Panel Power Route Isolation Update

این آپدیت برای جلوگیری از نشت UI و State مسیر اختصاصی `solar_panel_power` به سایر روش‌های محاسباتی اعمال شد.

## اصلاحات اصلی

- تشخیص مسیر توان پنل خورشیدی فقط با `shil:calculationMethod === "solar_panel_power"` انجام می‌شود.
- وجود دیتای قدیمی در `localStorage.shil:solarPanelPowerInput` دیگر باعث فعال شدن فرم‌ها و بانک‌های اختصاصی در سایر مسیرها نمی‌شود.
- صفحه پیکربندی سیستم برای مسیرهای دیگر از `panelCount`، `inverterSplitCount`، `acVoltageRoute` و `inverterPanelDistribution` مسیر توان پنل استفاده نمی‌کند.
- چکیده و خروجی نهایی فقط وقتی چکیده اختصاصی توان پنل را نشان می‌دهند که روش محاسبه واقعاً `solar_panel_power` باشد.
- بانک‌های اختصاصی مسیر توان پنل از مسیرهای دیگر جدا شدند؛ موتور محاسبات مرکزی همچنان مشترک و یکپارچه باقی می‌ماند.

## نکته تست

برای تست، ابتدا یک مسیر غیر از توان پنل مثل `power`، `current`، `energy` یا `equipment` را انتخاب کنید. نباید فرم‌های تقسیم پنل، MPPT اختصاصی، روزهای خودکفایی صفر پیش‌فرض، یا خروجی اختصاصی مسیر توان پنل در آن مسیرها فعال شود.
